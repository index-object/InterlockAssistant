# 窗口信息检测器设计

## 概述

开发一个独立的窗口检测/选择工具，用于收集目标窗口的详细信息，为后续自动捕获功能提供窗口识别依据。

## 功能需求

1. **窗口选择模式**
   - 用户点击"开始选择"按钮
   - 鼠标变为十字形状
   - 点击目标窗口后自动获取该窗口信息

2. **窗口基础信息收集**
   - 窗口句柄（HWND）
   - 窗口类名
   - 窗口标题
   - 进程名
   - 进程ID（PID）

3. **控件树探测**
   - 使用UI Automation API遍历窗口所有子控件
   - 显示控件类型、类名、名称
   - 重点识别输入框（Edit、RichEdit、QLineEdit等）

4. **输入框信息识别**
   - 列出所有输入框控件
   - 显示控件的AutomationId、Name、ClassName

5. **目标保存**
   - 用户确认窗口信息后，保存到配置文件
   - 后续自动捕获功能使用保存的配置

## 技术方案

- 使用 `uiautomation` 库（推荐）或直接使用 Windows UI Automation API
- 新建悬浮窗口（FloatingWindow）显示检测信息
- 使用现有的 `WindowPicker` 进行窗口选择

## 界面设计

- 悬浮窗口显示区域
  - 窗口基础信息区域
  - 控件树列表区域
- 操作按钮
  - "开始选择"按钮
  - "刷新"按钮
  - "保存目标"按钮
  - "关闭"按钮

## 接受标准

1. 用户点击目标窗口后，能正确获取窗口句柄
2. 能显示窗口的基础信息（类名、标题、进程名）
3. 能列出窗口下的主要控件
4. 能识别输入框控件并高亮显示
5. 保存的目标信息可被后续功能读取使用
